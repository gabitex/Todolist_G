Este proyecto es una API REST profesional que incluye manejo de Metadatos, Caché en servidor, optimización de tráfico mediante ETags y persistencia en la nube con MongoDB Atlas.

1. REQUISITOS PREVIOS
---------------------
* Tener instalado Node.js (Versión LTS recomendada).
* Si ya lo tiene, puede omitir este paso.

2. INSTALACIÓN
--------------
1. Descargue y extraiga el proyecto.
2. Abra una terminal en la carpeta raíz del proyecto e instale las dependencias (express, mongoose, dotenv, node-cache, etag):
   $ npm install

-----------------------------------

4. EJECUCIÓN DEL SERVIDOR
-------------------------
Inicie la aplicación con el comando:
$ npm start

Resultado esperado en consola:
- Servidor en http://localhost:300
- MongoDB conectado

5. PRUEBAS DE LA API (ENDPOINTS)
--------------------------------
La API utiliza una estructura de respuesta profesional con Metadatos (meta) y Cuerpo de datos (data). También implementa ETags para optimizar el ancho de banda (respuestas 304).

A. Consultar Tareas (GET)
- URL: http://localhost:300/listadetareas/appv1
- Nota: Puede probarse directamente en el navegador para observar los metadatos y el estado del caché.

B. Crear Tareas (POST)
- URL: http://localhost:300/listadetareas/appv1
- Body (JSON): {"tarea": "Nueva tarea"} o un Array [{"tarea": "T1"}, {"tarea": "T2"}].

C. Actualizar y Eliminar (PUT / DELETE)
- URL: http://localhost:300/listadetareas/appv1/ID_DE_LA_TAREA
- Para actualizar el estado o borrar, debe sustituir ID por el identificador único generado por MongoDB (visible en el GET).

-----------------------------------------------------------
CARACTERÍSTICAS ESPECIALES IMPLEMENTADAS:
-----------------------------------------------------------
* Metadatos: Cada respuesta GET incluye el total de registros y versión.
* Caché de Servidor: Uso de 'node-cache' para reducir consultas a la base de datos.
* Optimización de Tráfico: Uso de 'Cache-Control' y 'ETags' (Headers) para ahorro de ancho de banda.
* Estabilidad: Manejo de errores mediante bloques 'try-catch' para evitar caídas del servidor.
* Versionamiento: Implementación de rutas bajo el prefijo /appv1.
